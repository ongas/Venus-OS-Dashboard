# Copilot Process Requirements

**⚠️ MANDATORY: Follow this process for ALL changes to Venus-OS-Dashboard**

This file ensures Copilot agents always follow the correct release and deployment workflow.

## Pre-Work Checklist

- [ ] Read `DEPLOYMENT_CHANGE_MANAGEMENT.md` to understand the full workflow
- [ ] Verify this `.copilot-process.md` has not changed (if it has, re-read it)

## Change Workflow (ALL changes must follow this)

### 1. Make Changes
- Edit files in `dist/` (the source of truth for this project)
- **CRITICAL**: Use targeted edits only (never rewrite entire files)
- Preserve CRLF line endings — do NOT convert to LF
- Never use Python `open()`/`write()` as it breaks line endings

### 2. Run Release Script
```bash
./release.sh X.Y.Z
```

This automates:
- Cache-busting (`?v=X.Y.Z` params)
- CRLF normalization
- JS syntax validation (`node -c`)
- ESLint validation (`npm run lint`)
- .gz file regeneration
- Git staging of dist/ files

### 3. Commit & Tag
```bash
git commit -m "Release vX.Y.Z — description"
git tag vX.Y.Z
git push origin main vX.Y.Z
```

### 4. Create GitHub Release (DO NOT SKIP)
```bash
gh release create vX.Y.Z --title "vX.Y.Z" --notes "Detailed notes here"
```

**⚠️ CRITICAL**: HACS requires a GitHub Release object, not just a git tag.
Without this step, users will NOT see the update in HACS.

## Post-Release

- Verify release appears at: https://github.com/ongas/Venus-OS-Dashboard/releases
- HACS will auto-detect and notify users to update

## Editing Rules (Non-Negotiable)

- ✅ Use `edit` tool for targeted line replacements
- ✅ Use `sed -i` for multi-line patterns
- ✅ Validate syntax with `node -c dist/<name>.js`
- ❌ Never use Python `open()`/`write()` (breaks CRLF)
- ❌ Never use JavaScript private class fields (`#field`)
- ❌ Never rewrite entire files

## Common Mistakes to Avoid

| Mistake | Why Bad | Fix |
|---------|---------|-----|
| Skip `release.sh` and commit directly | Cache-busting params won't update, .gz files out of sync | Always run `release.sh` first |
| Create tag but no GitHub Release | HACS won't see the update | Always run `gh release create` |
| Convert CRLF to LF | HA Lovelace loader fails with "Configuration error" | Use `file` to verify CRLF, re-run `release.sh` |
| Rewrite entire files with Python | Breaks line endings | Use targeted edits only |
